# Тестирование проекта

## Запуск тестов

Для запуска всех тестов выполните команду:
```bash
./gradlew test
```

## Просмотр отчетов

После выполнения тестов HTML-отчет будет доступен по пути:
```
build/reports/tests/test/index.html
```

## Структура тестов

### Основные тестовые файлы:

1. **LearnWordsTrainerTest.kt** - тесты основного класса тренажера
   - Загрузка словаря
   - Расчет статистики
   - Генерация вопросов
   - Проверка ответов
   - Сброс прогресса
   - Обработка ошибок

2. **QuestionTest.kt** - тесты класса Question
   - Создание вопроса
   - Отображение в консоли
   - Обработка крайних случаев

3. **WordAndStatisticsTest.kt** - тесты моделей данных
   - Создание объектов Word
   - Создание объектов Statistics
   - Обработка специальных символов

## Покрытие тестами

Тесты покрывают следующие сценарии:
- ✅ Корректная загрузка словаря
- ✅ Обработка поврежденных файлов
- ✅ Расчет статистики
- ✅ Генерация вопросов (включая случайность)
- ✅ Проверка правильных и неправильных ответов
- ✅ Сброс прогресса
- ✅ Сохранение данных
- ✅ Обработка отсутствующих файлов
- ✅ Отображение вопросов в консоли

## Особенности реализации

- Используются временные файлы для тестов (`@TempDir`)
- Перехват консольного вывода для проверки отображения
- Тестирование случайных сценариев с использованием `requireNotNull`
- Проверка граничных случаев и обработки ошибок
